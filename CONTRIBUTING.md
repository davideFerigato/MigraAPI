# Contributing to MigraAPI

We welcome contributions! Please follow these guidelines.

## How to contribute

1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/amazing-feature`).
3. Make your changes, add tests if applicable.
4. Ensure the test suite passes: `python tests/test_migration.py`.
5. Commit with clear messages (Conventional Commits preferred).
6. Push and open a Pull Request.

## Code style

- Python: follow PEP 8.
- Markdown: wrap at 80 chars.
- JSON/YAML: use 2 spaces indentation.

## Reporting issues

Use the GitHub issue tracker. Include:
- Steps to reproduce
- Expected and actual behavior
- Relevant logs or screenshots
